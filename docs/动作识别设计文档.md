# 动作识别设计文档

## MPU6050 基本姿态信息

### ax、ay、az（加速度）

各轴方向的**线性加速度分量**，包含重力加速度在各轴上的投影。

- 量程：±2g
- 数值范围：-32768 ~ 32767
- 单位：g

在设备**静止或低速运动**时，主要反映的是**重力方向**，可用于姿态解算。

------

### gx、gy、gz（角速度）

各轴方向的**角速度**，反映设备的旋转快慢。

- 量程：±250 °/s
- 标度：131 LSB ≈ 1 °/s

> ⚠️ 角速度更适合短时动态变化，不适合作为“晃动”或“冲击”动作的主要判别依据。

------

### roll、pitch（姿态角）

- **roll**：绕 **X 轴** 的倾斜角度
- **pitch**：绕 **Y 轴** 的倾斜角度

在当前设备逻辑坐标系下：

- X 轴：左右方向
- Y 轴：前后方向
- Z 轴：上下方向

#### 姿态角计算方式（基于加速度）

```c
// Roll：左右倾斜（X）
float roll_acc = atan2(ax, az) * RAD_TO_DEG;

// Pitch：前后倾斜（Y）
float pitch_acc = atan2(
    ay,
    sqrt(ax*ax + az*az)
) * RAD_TO_DEG;
```

> 实际工程中，roll / pitch 通常与陀螺仪角速度通过**互补滤波**进行融合，以提高动态稳定性。

------

## 基准姿态（Reference Pose）

为适应不同玩家的自然持握姿势，引入**基准姿态**概念：

- 基准姿态记录某一时刻的 roll / pitch
- 后续姿态计算均以该姿态为零点

```text
rel_roll  = roll  - ref_roll
rel_pitch = pitch - ref_pitch
```

动作识别基于 **相对姿态（rel_roll / rel_pitch）** 进行。

------

## 动作判别方式

### 1️⃣ 左右倾斜（Tilt）

**判别信号**：相对 pitch
**动作类型**：持续姿态类

| 动作     | 判别条件                          |
| -------- | --------------------------------- |
| 向右倾斜 | `rel_pitch > +TILT_THRESHOLD_DEG` |
| 向左倾斜 | `rel_pitch < -TILT_THRESHOLD_DEG` |

- 推荐阈值：15° ~ 25°
- 动作在阈值范围内可持续存在
- 适用于方向控制、菜单选择等连续输入

------

### 2️⃣ 前后倾斜（可选扩展）

**判别信号**：相对 roll

| 动作 | 判别条件                         |
| ---- | -------------------------------- |
| 前倾 | `rel_roll > +ROLL_THRESHOLD_DEG` |
| 后倾 | `rel_roll < -ROLL_THRESHOLD_DEG` |

- 推荐阈值：15° ~ 25°
- 动作在阈值范围内可持续存在
- 适用于方向控制、菜单选择等连续输入

------

### 3️⃣ 晃动（Shake）

**判别信号**：加速度模长

```c
float acc_norm = sqrt(ax*ax + ay*ay + az*az);
```

- 静止状态：`acc_norm ≈ 1.0 g`
- 晃动状态：`acc_norm` 明显偏离 1g

**判别条件：**

```c
fabs(acc_norm - 1.0f) > SHAKE_ACC_THRESHOLD
```

| 参数                  | 说明                   |
| --------------------- | ---------------------- |
| `SHAKE_ACC_THRESHOLD` | 推荐 0.30 ~ 0.45 g     |
| 动作类型              | 瞬时冲击类             |
| 特点                  | 与旋转解耦，抗误触性好 |

------

## 动作去重与冷却机制

为防止同一动作被连续触发，引入**冷却时间与去重逻辑**：

- 同一动作在冷却时间内只触发一次
- 动作消失后允许再次触发

```c
#define MOTION_COOLDOWN_MS 300
```

## 动作判别方式汇总表

| 动作 | 判别信号  | 类型     |
| ---- | --------- | -------- |
| 左倾 | rel_pitch | 持续姿态 |
| 右倾 | rel_pitch | 持续姿态 |
| 前倾 | rel_roll  | 持续姿态 |
| 后倾 | rel_roll  | 持续姿态 |
| 晃动 | acc_norm  | 冲击动作 |

------

## 设计说明（重要）

- 每个动作**只依赖一种主判别信号**
- 不混合使用加速度 / 姿态 / 角速度
- 有利于：
  - 调参
  - 排错
  - 后期游戏逻辑扩展

# 一、你现在读到的 6 个数值是哪些？

```cpp
mpu6050_raw_t {
    int16_t ax, ay, az;   // 加速度计
    int16_t gx, gy, gz;   // 陀螺仪
}
```

它们来自 MPU6050 内部的 **两种传感器**：

| 传感器                    | 作用                    |
| ------------------------- | ----------------------- |
| Accelerometer（加速度计） | 感知“倾斜 + 线性加速度” |
| Gyroscope（陀螺仪）       | 感知“旋转角速度”        |

------

# 二、加速度计 ax / ay / az 是什么？

### 1️⃣ 本质

👉 **测量的是某一轴方向上的“合加速度”**

包括：

- 重力加速度（1g）
- 人为移动、晃动的加速度

------

### 2️⃣ 单位 & 原始值含义

你现在读的是 **原始 ADC 值**（raw）

在默认配置（±2g）下：

| 数值   | 含义 |
| ------ | ---- |
| 16384  | 1g   |
| 0      | 0g   |
| -16384 | -1g  |

📌 换算公式：

```cpp
加速度(g) = raw / 16384.0
```

------

### 3️⃣ 各轴在“静止”时应该看到什么？

假设模块平放、正面朝上：

| 轴   | 典型值   |
| ---- | -------- |
| ax   | ≈ 0      |
| ay   | ≈ 0      |
| az   | ≈ +16384 |

👉 **因为重力指向 Z 轴**

------

### 4️⃣ 能用来干嘛？

✔ 判断“倾斜方向”
✔ 判断“是否被甩动 / 摇晃”
✔ 简单体感（左右 / 前后）

❌ 不适合：

- 精确角度
- 快速旋转（会抖）

------

# 三、陀螺仪 gx / gy / gz 是什么？

### 1️⃣ 本质

👉 **测量角速度（转得有多快）**

不是角度，是：

```
°/s（每秒转多少度）
```

------

### 2️⃣ 原始值单位

默认配置（±250°/s）：

| 数值 | 含义  |
| ---- | ----- |
| 131  | 1°/s  |
| 0    | 静止  |
| -131 | -1°/s |

📌 换算公式：

```cpp
角速度(°/s) = raw / 131.0
```

------

### 3️⃣ 静止时应该是什么？

| 轴   | 值   |
| ---- | ---- |
| gx   | ≈ 0  |
| gy   | ≈ 0  |
| gz   | ≈ 0  |

📌 实际会看到：

```
-10 ~ +10 之间波动（正常）
```

------

### 4️⃣ 能用来干嘛？

✔ 判断“旋转方向”
✔ 快速动作检测
✔ 手柄转动 / 甩动

❌ 不适合：

- 单独长期算角度（会漂）

------

# 四、为什么要“两个一起用”？

| 传感器   | 优点       | 缺点     |
| -------- | ---------- | -------- |
| 加速度计 | 稳定、不漂 | 动作时乱 |
| 陀螺仪   | 反应快     | 会漂     |

👉 **体感的核心公式**：

> **加速度管“稳态”
> 陀螺管“动态”**

所以你以后一定会看到：

```
互补滤波 / 卡尔曼滤波
```

------

# 五、结合你这个“体感游戏机”怎么用？

### 🎮 直觉映射（新手最适合）

| 动作     | 用哪个  |
| -------- | ------- |
| 左右倾斜 | ax      |
| 前后倾斜 | ay      |
| 快速甩手 | gx / gy |
| 转向     | gz      |

例如：

```cpp
if (imu.ax > 8000)  // 向右倾
    move_right();

if (abs(imu.gx) > 15000) // 快速甩动
    attack();
```

# 六、一个“立刻有感觉”的小实验（强烈推荐）
